def research_node(state: ResearchState) -> dict:
    llm   = ChatOpenAI(model="gpt-4o-mini", temperature=0)
    agent = create_react_agent(
        llm,
        tools=[TavilySearch(max_results=5)],
        prompt=RESEARCH_SYSTEM_PROMPT,       # instructs agent to run 4‚Äď6 diverse queries

The ReAct agent autonomously decides *how many* searches to run and from *which angle* ‚ÄĒ
basics, recent developments, statistics, expert opinions, case studies, etc.
Sources are extracted from Tavily tool-call messages and deduplicated by URL.

</details>

<details>
<summary><strong>‚úč human_review_node</strong> ‚ÄĒ click to expand</summary>

```python
def human_review_node(state: ResearchState) -> dict:
    approved = interrupt(state["sources"])   # ‚Üź execution PAUSES here
    return {"approved_sources": approved}    # ‚Üź execution RESUMES here
```

`interrupt()` raises a special LangGraph exception that:
1. Serializes the complete graph state into `MemorySaver`
2. Returns control to the calling code (Streamlit)

When the user clicks **Approve & Summarize**, Streamlit calls:
```python
graph.invoke(Command(resume=selected_sources), config=config)
```
LangGraph restores the frozen state and injects the user's selection.

</details>

<details>
<summary><strong>‚úćÔłŹ summarize_node</strong> ‚ÄĒ click to expand</summary>

```python
def summarize_node(state: ResearchState) -> dict:
    sources_text = "\n\n".join(
        f"Title: {s['title']}\nURL: {s['url']}\nContent: {s['content']}"
        for s in state["approved_sources"]
    )
    prompt = (
        f'Write a comprehensive summary about "{state["topic"]}" based on these sources:\n\n'
        f"{sources_text}\n\n"
        "Structure your response with:\n"
        "1. Overview\n2. Key Points\n3. Different Perspectives\n4. Conclusions"
    )
    response = llm.invoke([HumanMessage(content=prompt)])
    return {"summary": response.content}
```

## ūüďĖ How Human-in-the-Loop Works

The key concept is LangGraph's **interrupt / resume** pattern ‚ÄĒ the graph can pause mid-execution, wait for human input, and then continue from exactly where it left off.

```
Standard agent:   node-A ‚ĒÄ‚ĒÄ‚Ėļ node-B ‚ĒÄ‚ĒÄ‚Ėļ node-C ‚ĒÄ‚ĒÄ‚Ėļ done
                                 ‚ÜĎ
NotebookLM Mini:          PAUSES HERE ‚Źł
                          Streamlit shows sources to user
                          User selects what to keep
                          Graph resumes ‚Ė∂ with approved list
```

### Step-by-step flow

```
1. User types topic  ‚ĒÄ‚ĒÄ‚Ėļ  Streamlit calls graph.invoke({"topic": ...})

2. research_node runs   ‚ĒÄ‚ĒÄ‚Ėļ  Agent queries Tavily 4‚Äď6 times
                              Sources collected & deduplicated

3. human_review_node    ‚ĒÄ‚ĒÄ‚Ėļ  interrupt(sources) is called
                              ‚Źł Graph FREEZES ‚ÄĒ state saved to MemorySaver
                              graph.invoke() returns to Streamlit

4. Streamlit renders    ‚ĒÄ‚ĒÄ‚Ėļ  Review page displayed to user
   review page               User checks/unchecks source cards

5. User clicks          ‚ĒÄ‚ĒÄ‚Ėļ  Streamlit calls:
   "Approve"                 graph.invoke(Command(resume=selected), config=cfg)

6. Graph RESUMES        ‚ĒÄ‚ĒÄ‚Ėļ  human_review_node returns {"approved_sources": selected}

7. summarize_node runs  ‚ĒÄ‚ĒÄ‚Ėļ  GPT-4o-mini generates structured summary
                              from only the approved sources

8. Done page            ‚ĒÄ‚ĒÄ‚Ėļ  Summary + source cards displayed
```

> **Why `MemorySaver`?** Streamlit re-renders the entire Python script on every user interaction. Without the checkpointer persisting state, the graph would restart from scratch on every click. `MemorySaver` acts as a session-scoped database that keeps the frozen graph state alive between HTTP requests.
